## TK-2026-0022（questions-takken.js）

### 1. 現在の問題内容の要約

- **topic**：手付金の額の制限
- **形式**：誤っているものはどれか（誤り選択）
- **correct**：1（options[1]が誤り）

| # | 選択肢 | 正誤 |
|---|--------|------|
| 0 | 手付金の額は代金の20%を超えることができない | 正しい |
| 1 | 20%を超える手付金を定めた場合、定め自体がすべて無効となる | **誤り（正解）** |
| 2 | 受領した手付金は別段の定めがない限り解約手付の性質を持つ | 正しい |
| 3 | 買主が宅建業者の場合、20%超でも宅建業法違反とならない | 正しい |

- **explanation**：「超過部分のみ無効（全部無効ではない）」

### 2. C判定になった理由

類似問題 **TK-2026-0195**（正しいものはどれか）の正解選択肢が  
「超過部分のみ無効・2割以内は有効」であり、本問のoptions[1]（誤り）と  
**正誤が逆なだけで同一の知識（超過部分のみ無効）を正答条件とする**。  
問題形式は異なるが、受験生が正答に使う知識に差がない。

### 3. 修正方針案A（options[1]を手付保全措置の誤記述に差し替え）

options[1]のみ変更し、「手付金等の保全措置」の誤った説明に差し替える。  
question・options[0][2][3]・correctは変更しない。

```
【現在のoptions[1]】
「代金の20%を超える手付金を定めた場合、その手付金の定め自体がすべて無効となる。」

【案Aのoptions[1]】
「宅地建物取引業者が自ら売主となる場合、完成物件・未完成物件を問わず、
手付金等の受領前に保全措置を講じなければならない。」
（← 誤り：完成物件は代金5%かつ1000万円以下なら保全措置不要）
```

- correct：**1のまま変更不要**（新options[1]も誤りのため）
- explanation：「超過部分のみ無効」に加え、「完成物件の保全措置要件（5%超または1000万円超）」を追記

**差別化の効果**  
本問 → 手付金保全措置の要件を問う  
TK-0195 → 超過部分のみ無効の原則を問う  
→ 異なる論点として共存可能

**実装リスク（案A）**  
- 低：options[1]の1行差し替えのみ。correct変更なし。
- 新options[1]の誤記述内容が宅建試験の正確な知識に反しないか要確認
- 「未完成物件の保全措置」は代金5%超または1000万円超の場合に必要（完成物件は10%超または1000万円超）という数値の確認が必須

### 4. 修正方針案B（options[1]を手付放棄・倍返し解除の誤記述に差し替え）

「解約手付による契約解除のタイミング」を誤った記述として差し替え。

```
【案Bのoptions[1]】
「買主が手付を放棄することによる解除は、相手方が契約の履行に着手した後であっても
行うことができる。」（← 誤り：相手方の履行着手後は解除不可）
```

- correct：**1のまま変更不要**
- explanation：「履行着手後は解約手付による解除不可」を追記

**差別化の効果**  
本問 → 解約手付による解除可能タイミングを問う  
TK-0195 → 超過部分のみ無効の原則を問う  
→ 異なる論点として共存可能

**実装リスク（案B）**  
- 低：options[1]の1行差し替えのみ。correct変更なし。
- options[2]（「受領した手付金は解約手付の性質を持つ」）との論点近接に注意

### 5〜8. 変更項目・差別化サマリー

| 変更箇所 | 案A | 案B |
|---------|-----|-----|
| question | 変更不要 | 変更不要 |
| options[1]のみ | 変更あり | 変更あり |
| correct | 変更不要（1のまま） | 変更不要（1のまま） |
| explanation | 追記あり | 追記あり |
| 類似問題との差別化 | ◎ 完全に別論点 | ○ 別論点だがoptions[2]と近接 |

### 9. 実装リスク

- **低リスク**：options[1]の1行変更のみ。問題構造（question・correct・他options）は保持される
- 注意：新選択肢の法的正確性（保全措置の数値要件・履行着手の定義）の確認が必要

### 10. THINK推奨案

**案A を推奨**  
理由：「手付金保全措置の要件」はTK-0195と完全に別論点であり、かつoptions[2]（解約手付の性質）との重複もない。案Bは選択肢[2]と論点が近接しすぎるリスクがある。

## TK-2026-0122（questions-rights.js）

### 1. 現在の問題内容の要約

- **topic**：保証と連帯保証
- **形式**：正しいものはどれか（正解選択）
- **correct**：2（options[2]が正しい）

| # | 選択肢 | 正誤 |
|---|--------|------|
| 0 | 保証契約は口頭の合意のみでも有効に成立する | 誤り |
| 1 | 連帯保証人には催告の抗弁権が認められている | 誤り |
| 2 | 保証人には催告の抗弁権あり・連帯保証人にはない | **正しい（正解）** |
| 3 | 保証人・連帯保証人いずれも分別の利益がある | 誤り |

- **explanation**：「保証人には催告・検索・分別の利益あり、連帯保証人にはなし。保証契約は書面が必要。」

### 2. C判定になった理由

類似問題 **TK-2026-0167** の正解選択肢が  
「連帯保証人には催告の抗弁権も検索の抗弁権も認められない」であり、  
本問options[2]と**正解の核心が同一**（連帯保証人には抗弁権なし）。  
どちらも「正しいものを選ぶ」形式で、正解を導くための知識が重複している。

### 3. 修正方針案A（options[1]を連帯保証の絶対効の記述に差し替え）

options[1]を「連帯保証人に対する履行請求の効力」に差し替え、  
かつoptions[2]の正解内容を「保証人の付従性」に変更する。

```
【案Aのoptions[1]】
「連帯保証人に対して行われた履行の請求は、主たる債務者に対してその効力を生じない。」
（← 誤り：連帯保証人への請求は主債務者に絶対効として効力が及ぶ）

【案Aのoptions[2]】
「保証債務は主たる債務に付従し、主たる債務が消滅すれば保証債務も消滅する。」
（← 正しい：付従性）
```

- correct：**2のまま変更不要**（options[2]を新しい「正しい内容」にするため）
- explanation：「付従性」に関する説明に改訂

**差別化の効果**  
本問 → 保証の付従性と連帯保証の絶対効を問う  
TK-0167 → 催告の抗弁権・検索の抗弁権の有無を問う  
→ 異なる論点として共存可能

**実装リスク（案A）**  
- **中リスク**：options[1][2]の両方を変更するため、内容の整合性確認が必要
- correct は2のままだが、options[2]の内容が変わるため説明文全体の見直しが必要
- 「絶対効の範囲（履行請求・相殺・更改・混同）」の知識確認が必要

### 4. 修正方針案B（question自体を連帯保証の絶対効専用問題に再設計）

問題文ごと変更し、「連帯保証の絶対効・相対効」に特化した問題に再設計する。

```
【案Bのquestion】
「民法における連帯保証に関する次の記述のうち、誤っているものはどれか。」

【案Bのoptions例】
0: 連帯保証人には催告の抗弁権も検索の抗弁権も認められない（正しい → 誤り問題なので誤りではない）
1: 連帯保証人に対して行われた履行の請求は、主たる債務者に対してその効力を生じない（誤り → correct）
2: 連帯保証契約は書面によらなければ効力を生じない（正しい）
3: 連帯保証人が主たる債務について相殺権を行使すると、その効力は主たる債務者にも及ぶ（正しい）
```

- correct：変更が必要（新しい「誤り」の位置に合わせる）
- explanation：連帯保証の絶対効・相対効の説明に全面改訂

**差別化の効果**  
本問 → 連帯保証の絶対効・相対効を問う  
TK-0167 → 催告・検索の抗弁権を問う  
→ 完全に異なる論点

**実装リスク（案B）**  
- **高リスク**：question・options全体・correct・explanationの全面変更が必要
- 絶対効の範囲（履行請求・相殺・更改・混同・時効の完成猶予など）の正確な知識確認が必須
- 誤り問題と正解問題の切り替えも伴う

### 5〜8. 変更項目・差別化サマリー

| 変更箇所 | 案A | 案B |
|---------|-----|-----|
| question | 変更不要 | 変更あり |
| options | [1][2]変更 | 全面変更 |
| correct | 変更不要（2のまま） | 変更あり |
| explanation | 改訂あり | 全面改訂 |
| 類似問題との差別化 | ◎ 別論点 | ◎ 完全別論点 |

### 9. 実装リスク

- 案A：中リスク（2選択肢変更 + explanation改訂、correct不変）
- 案B：高リスク（全面再設計、知識確認の範囲が広い）

### 10. THINK推奨案

**案A を推奨**  
理由：correct（2）を変えずに2選択肢のみ変更することで、既存問題構造を最大限保ちつつ差別化できる。案Bは高品質だが全面再設計コストが高く、誤記リスクも高い。案Aで「付従性と絶対効の2論点を組み合わせた問題」として再設計すれば十分に差別化できる。

## TK-2026-0052（questions-laws.js）

### 1. 現在の問題内容の要約

- **topic**：区域区分（市街化区域・市街化調整区域）
- **形式**：正しいものはどれか（正解選択）
- **correct**：3（options[3]が正しい）

| # | 選択肢 | 正誤 |
|---|--------|------|
| 0 | 市街化調整区域は「すでに市街地を形成している区域…」（← 市街化区域の定義を逆に記述） | 誤り |
| 1 | 市街化区域は市街化を抑制すべき区域（← 逆） | 誤り |
| 2 | すべての都市計画区域で区域区分を必ず定めなければならない | 誤り |
| 3 | 市街化区域は「すでに市街地を形成している区域」＋「10年以内に計画的に市街化を図るべき区域」 | **正しい（正解）** |

- **explanation**：「市街化区域の定義。市街化抑制は調整区域。区域区分は必須でない。」

### 2. C判定になった理由

類似問題 **TK-2026-0123** の正解選択肢（options[0]）が  
「市街化区域は『すでに形成している区域』＋『10年以内に計画的に』」であり、  
本問options[3]と**一字一句ほぼ同じ**。  
両問とも「正しいものを選ぶ」形式で、正解を導くための知識が完全に一致している。

### 3. 修正方針案A（questionと選択肢を「用途地域」にフォーカスして再設計）

question・options全体を「用途地域」にテーマ変更し、都市計画法の別論点をカバーする。

```
【案Aのquestion】
「都市計画法における用途地域に関する次の記述のうち、正しいものはどれか。」

【案Aのoptions例】
0: 用途地域は、都市計画区域外でも必要に応じて指定できる（誤り：都市計画区域外には指定できない）
1: 準都市計画区域においても、市街化区域と同様に13種類の用途地域をすべて指定することができる（誤り：準都市計画区域では用途地域は指定できない）
2: 市街化調整区域においても、必要に応じて用途地域を定めることができる（誤り：原則として定めない）
3: 市街化区域については、少なくとも用途地域を定めるものとされている（正しい → correct: 3）
```

- correct：**3のまま変更不要**（options[3]を新しい「正しい内容」にするため、上記設計の場合）
- explanation：「市街化区域には必ず用途地域を定める」に全面改訂

**差別化の効果**  
本問 → 用途地域の指定要件・指定範囲を問う  
TK-0123 → 市街化区域の定義・区域区分の必要性を問う  
→ 完全に異なる論点

**実装リスク（案A）**  
- **中リスク**：question・options全体の変更が必要。correct位置は変更不要に設計可能
- 「用途地域の指定できる区域の範囲（都市計画区域内の市街化区域・非線引き区域のみ等）」の正確な知識確認が必須
- 新選択肢の誤記述内容が宅建試験の正確な知識に反しないか要確認

### 4. 修正方針案B（options[3]を「準都市計画区域の特性」に差し替え、正解位置を変更）

questioは「区域区分」のまま維持し、options[3]（現在の正解）を  
「準都市計画区域に関する正しい記述」に差し替える。

```
【案Bのoptions[3]】
「準都市計画区域は、都市計画区域外において、指定都市等の申出を基に、
都道府県が指定する区域である。」（正しい）
```

また、options[0]〜[2]のうちひとつを「区域区分の選択制」に関する正しい記述に変更し、  
正解位置を変える。

- correct：**変更が必要**（正解位置が変わるため）

**差別化の効果**  
本問 → 準都市計画区域の位置付けを問う  
TK-0123 → 市街化区域の定義を問う  
→ 別論点として共存可能だが差別化は中程度

**実装リスク（案B）**  
- **高リスク**：correct変更が必要。options内の正誤バランス再設計が必要
- 「区域区分」という同じquestionのまま正解を変えると、問題の整合性確認コストが高い

### 5〜8. 変更項目・差別化サマリー

| 変更箇所 | 案A | 案B |
|---------|-----|-----|
| question | 変更あり（用途地域へ） | 変更不要 |
| options | 全面変更 | options[3]等変更 |
| correct | 変更不要（3のまま・設計次第） | **変更あり** |
| explanation | 全面改訂 | 改訂あり |
| 類似問題との差別化 | ◎ 完全別論点 | △ 差別化は中程度 |

### 9. 実装リスク

- 案A：中リスク（全面書き直しだが、correct位置を3に固定した設計で安定化できる）
- 案B：高リスク（correct変更が必要で、整合性確認コストが最も高い）

### 10. THINK推奨案

**案A を推奨**  
理由：「用途地域の指定要件」は宅建試験頻出論点でありTK-0123と完全に別論点。correct位置（3）を変えない設計で案Aを実装すれば、correct変更リスクがなく最も安全。案Bはcorrect変更が必要なため整合性リスクが高い。
