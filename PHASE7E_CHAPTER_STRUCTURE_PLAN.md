## 1. 現在のチャプター表示の仕組み

### 定数・関数の所在

| 要素 | ファイル | 場所 |
|---|---|---|
| `CHAPTER_SIZE = 25` | script.js | line 1（先頭） |
| `renderCategoryChapters()` | script.js | line 500–551 |
| `startQuiz(mode, category, chapterStart)` | script.js | line 136 |

### チャプター生成ロジック（現在）

```js
// renderCategoryChapters() 内
for (let start = 0; start < allQs.length; start += CHAPTER_SIZE) {
  const chNum = Math.floor(start / CHAPTER_SIZE) + 1;
  // ↓ Chapter 1 / Chapter 2 / Chapter 3 と表示
  `<div class="chapter-title">Chapter ${chNum}</div>`
}
```

- カテゴリ内の問題を `start=0`, `25`, `50` でスライスするだけ
- 表示名は機械的に `Chapter 1 / 2 / 3` と生成される
- 問題データ側に章情報は**ない**（章はscript.jsの計算のみで決まる）
- `startQuiz` には `chapterStart`（0 / 25 / 50）が渡され、スライス開始位置として使う

## 2. 問題点（有識者レビューの要点）

### 現在の表示

```
⚖️ 権利関係
  [全問まとめて] 75問
  [Chapter 1]   25問 ・ 学習済み○問
  [Chapter 2]   25問 ・ 学習済み○問
  [Chapter 3]   25問 ・ 学習済み○問
```

### 何が問題か

- 「Chapter 1」「Chapter 2」「Chapter 3」という名前から、**どの論点を勉強するのかが分からない**
- 特に権利関係・法令上の制限は論点が多く、チャプター選択の意味がない
- 学習者が「どこから始めるべきか」「どこが苦手か」を判断できない
- 有識者レビュー：「チャプター1・2・3では何を学ぶか分かりにくい」

## 4. 推奨案

### **A案を採用する**（C案を補完として追加）

#### 理由

1. **最小変更で最大効果**: script.js の2箇所のみ変更でチャプター選択画面の視認性が大きく向上する
2. **questions-*.js を変更しない**: 問題データに手を入れないためverify-questions.ps1 の結果に影響しない
3. **localStorageへの影響ゼロ**: chapterStart（0/25/50）は変わらないため、学習履歴に影響しない
4. **topic フィールドが既に存在している**: 各問題に `topic` フィールドが付いているため、どの論点が各チャプターに含まれるかは既に把握済み
5. **500問化への拡張が容易**: CHAPTER_LABELS にエントリを追加するだけで対応できる

#### 実装後の表示イメージ

## 付記：調査で確認できた重要事実

1. **`topic` フィールドは全300問に既に存在する** — questions-*.js の各問題オブジェクトに `topic: "..."` が付いている（verify-questions.ps1の必須フィールドではないが、全問題に付いている）
2. **チャプター分割はscript.js側の計算のみで決まる** — questions-*.js に章情報はない
3. **chapterStartの値（0/25/50）はdata-value属性でHTMLに埋め込まれ、startQuizに渡される** — ラベルを変えてもこの値は変わらない
4. **CHAPTER_SIZE=25 はscript.js先頭の定数** — 将来変更する場合はここ1行のみ変更
